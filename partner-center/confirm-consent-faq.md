## <a href="" id="processfaq"></a>Proceso

**¿Qué es el proceso de nuevo para confirmar la aceptación del cliente del contrato de nube de Microsoft (MCA)?**

Microsoft se compromete a nuestros clientes y partners mantendrás en las normas. Como partner en el programa de proveedor de soluciones en la nube (CSP) de Microsoft, eres responsable de términos revisión de Microsoft y condiciones, incluido el uso aceptable de los servicios con los clientes y productos de Microsoft. Para ayudar a los partners en cumplimiento de requisitos mejor y para garantizar la transparencia, estamos habilitando un nuevo proceso que requiere que los partners CSP confirmar que sus clientes han aceptado los términos MCA. A partir del 7 de agosto de 2018, hemos introducido un nuevo campo en el panel del centro de partners interfaz de usuario, así como una API del centro de partners para los partners para proporcionar esta información. Inicialmente estará opcional proporcionar esta confirmación, sin embargo, será necesario a partir de 7 de noviembre de 2018. 

**¿Cuándo se ve este requisito en vigor?**

Los partners pueden comenzar confirmar aceptación tan pronto como la funcionalidad está disponible en el 7 de agosto de 2018. A partir de 7 de noviembre de 2018, los usuarios de panel del centro de Partners deben para confirmar la aceptación del cliente de la MCA de todas las transacciones nuevos. 22 de marzo de 2019 será necesarios hacer lo mismo a los usuarios de API del centro de partners. La fecha límite para que los usuarios de la API se ha ampliado para permitir que adicionales tiempo de implementación técnica.

**NUEVO | ¿Microsoft valida la precisión de la información del cliente proporcionada por el partner?**

Microsoft valida que la dirección de correo electrónico del cliente proporcionada es del formato correcto. Es responsabilidad del partner para proporcionar información precisa y corregir cualquier información que se encuentra a ser incorrecta. 

**NUEVO | ¿Cómo puede un partner demostrar la aceptación del cliente de MCA, si está firmado, documentada, digitalizado, o en papel?**

El partner es responsable de garantizar que el cliente ha aceptado los términos de la MCA y para administrar su propio proceso y registros. Mediante el proceso de panel del centro de partners o API del centro de partners, el partner confirma que el cliente ha aceptado los términos de la MCA.

**NUEVO | ¿Cómo Microsoft comprobará a que finalice el proceso de confirmación para un cliente?**

A partir de 7 de noviembre de 2018, cuando un partner de CSP inicia una transacción con el panel del centro de partners para un cliente nuevo o existente, los sistemas de Microsoft automáticamente comprobará si la información de confirmación de cliente está presente en el centro de partners Panel. Si no está presente, la transacción no podrá realizarse. 

A partir 22 de marzo de 2019, cuando un partner de CSP inicia una transacción de un cliente nuevo o existente mediante la API del centro de partners, los sistemas de Microsoft automáticamente comprobará si la información de confirmación de cliente está presente. Si no está presente, la transacción no podrá realizarse. 

**NUEVO | ¿Notificación se envía a los clientes cuya dirección de correo electrónico se proporciona en el proceso de confirmación?**

No hay ningún proceso de notificación.

## <a href="" id="policyfaq"></a>Directiva

**¿Por qué es Microsoft requerir que los partners confirmar que los clientes han aceptado el MCA?** 

El contrato de nube de Microsoft (MCA) protege los partners, los clientes y Microsoft al ofrecer los derechos de uso del cliente, los términos y condiciones concedidas para los pedidos de productos y servicios de Microsoft Cloud. 

Ejecución correcta de la MCA protege nuestros socios, los clientes y Microsoft garantizando alineación mutua en muchos temas importantes incluidos pero sin limitarse a seguridad, privacidad y protección de datos. El proceso de confirmación garantiza la transparencia y la alineación y permite a Microsoft para ayudar a los clientes que necesitan para responder rápidamente a las consultas reglamentarias mejor.

**¿Qué información de cliente es necesaria para completar el proceso de confirmación?**
La siguiente información es necesaria respecto a la persona que ha aceptado el acuerdo:
-   Fecha de aceptación del contrato
-   Nombre de la primero y último
-   Dirección de correo electrónico
-   Número de teléfono (opcional) 

**NUEVO | ¿Qué sucede si no se cumple la fecha de vencimiento?**

Los partners pueden comenzar confirmar aceptación tan pronto como la funcionalidad está disponible en el 7 de agosto de 2018. A partir de 7 de noviembre de 2018, los usuarios de panel del centro de Partners deben para confirmar la aceptación del cliente de la MCA de todas las transacciones nuevos. 22 de marzo de 2019 será necesarios hacer lo mismo a los usuarios de API del centro de partners.

En el momento de la aplicación, no se permitirá nuevas compras para un cliente existente o cambios para acomodar el recuento de suscripciones existentes a hasta que se ha confirmado la aceptación del cliente. Para un nuevo cliente, no se permitirá transacciones hasta que se ha confirmado la aceptación. La renovación automática no se ve afectada por este requisito.

**¿Las regiones de este requisito se aplica a?**

Confirmación de aceptación del cliente para el MCA es necesario para todas las regiones en la nube pública de Microsoft. Confirmación de aceptación del cliente no es compatible con las nubes nacionales (incluido Microsoft Cloud for US Government y Microsoft Cloud Alemania).

**¿Los partners que son responsables de confirmar la aceptación del cliente para el MCA?**

Efectiva el 7 de noviembre de 2018, proveedores indirectos de proveedor de servicios en la nube (CSP) y los partners de CSP de facturación directa son responsable de confirmar la aceptación del cliente y puede usar la API del centro de partners o el panel del centro de partners para hacerlo. Los proveedores indirectos tengas que primero recopilar la información necesaria de sus distribuidores indirectos y, a continuación, hacer que esté disponible a Microsoft a través de la API del centro de partners o el panel del centro de partners. Por contrato, es el proveedor indirecto que tenga un enlace con Microsoft para garantizar que los clientes finales de los revendedores de su CSP han aceptado para el MCA de contrato. 

**¿Qué escenarios de cliente requieren confirmación de aceptación de cliente de la MCA?**

La aceptación del contrato de Microsoft en la nube debe confirmarse cuando quieren que los partners de CSP hacer que las transacciones siguientes:
1.  Un nuevo cliente complete nuevas compras.
2.  Un cliente existente complete nuevas compras.
3.  Número de puestos de actualizaciones de un cliente existente en las suscripciones existentes

**¿Puedo confirmar la aceptación para los clientes existentes independientes de sus nuevas compras o adiciones de número de puestos?**

Sí. Puede confirmarse aceptación del cliente para todos los clientes cubiertos por el MCA.

**¿Cuáles son los requisitos para los clientes que tienen varios partners CSP?**

Confirmación de aceptación se ha confirmado el Partner. Si un cliente realiza transacciones con varios partners CSP, será necesario cada partner para confirmar la aceptación.

**Soy un revendedor indirecto de CSP trabajar con proveedores indirectos de CSP. ¿Cuál es mi responsabilidad?**

Por contrato, es el proveedor indirecto que tenga un enlace con Microsoft para garantizar que el cliente final ha aceptado el MCA de contrato. Sin embargo, los proveedores indirectos pueden tener disposiciones independientes con revendedores indirectos para recopilar la aceptación del cliente. En este caso, los proveedores indirectos que primero recopilar la información de aceptación de cliente obligatorio de sus distribuidores indirectos y, a continuación, hacer que esté disponible a Microsoft a través de la API del centro de partners o el panel del centro de partners. Este proceso puede admitir proporcionando rápidamente la información necesaria para los proveedores indirectos tras su solicitud.

**Soy un proveedor indirecto de CSP. ¿Es necesario coordinar con mi revendedor indirecto de CSP para estar preparado?**

Se recomienda proveedores indirectos que necesitan para recopilar la información de aceptación de cliente obligatorio de sus revendedores indirectos para comunicar este requisito a sus revendedores indirectos inmediatamente a la distribución y la recepción de esta información de coordenadas.

**Cómo ¿explicar a los clientes que debe confirmar que su aceptación del MCA?**

Ejecución correcta de la MCA protege Microsoft, sus clientes y partners garantizando alineación mutua en muchos temas importantes incluidos pero sin limitarse a seguridad, privacidad y protección de datos. El proceso de confirmación garantiza la transparencia y la alineación y permite a Microsoft para ayudar a los clientes que necesitan para responder rápidamente a las consultas reglamentarias mejor.

**NUEVO | Si un cliente funciona con un único proveedor indirecto de CSP, pero se completen las transacciones con varios revendedores indirectos de CSP, ¿necesita cada revendedor indirecto de CSP proporcionar el proveedor indirecto de CSP con la información de confirmación?**

El proveedor indirecto de CSP es responsable de proporcionar la confirmación de aceptación del cliente MCA a Microsoft. Para cada cuenta de cliente, solo una confirmación puede registrarse en el panel del centro de partners al mismo tiempo. 

Es responsable de confirmar que ha realizado una persona de contacto de cliente responsable de comprobación de antecedentes el MCA por lo tanto, independientemente del número de revendedores indirectos de CSP que realizar transacciones con el proveedor indirecto de CSP.

**NUEVO | ¿Que debe ser la persona autorizada para confirmar la aceptación del MCA dentro de la organización del cliente?**

Es responsabilidad del partner obtener aceptación de cliente de la MCA de una persona dentro de la organización del cliente. Es responsabilidad del cliente para determinar quién está autorizado para proporcionar la aceptación. 

**NUEVO | ¿Qué versión de la MCA son partners responsables de confirmar la aceptación del cliente de? ¿Los partners deben indicar qué versión de la MCA se ha aceptado por el cliente?**

Los partners son responsables de usando el MCA más reciente al realizar las transacciones de CSP con un cliente nuevo. Los Partners deben confirmar la aceptación de lo más reciente MCA cuando renovar, cambiar o completar nuevas transacciones con un cliente existente. 

En el caso de que se actualiza el MCA antes de la renovación, los partners no es necesario tener el cliente volverá a obtener aceptación de cliente de la MCA inmediatamente, pero deben confirmar consentimiento para el MCA antes de la renovación. Sin embargo, puede que el cliente acepte términos nuevos antes de la renovación para sacar partido de nuevos y mejorados de los términos de la MCA.

**NUEVO | Soy un revendedor indirecto de CSP. Si la grabación de aceptación de cliente de la MCA en los registros de mi empresa, ¿necesito también proporcionar esta información a través del panel del centro de partners o la API del centro de partners?**

Por contrato, es el proveedor indirecto que tenga un enlace con Microsoft para garantizar que el cliente final ha aceptado el MCA de contrato. Sin embargo, los proveedores indirectos pueden tener disposiciones independientes con revendedores indirectos para recopilar la aceptación del cliente. En este caso, los proveedores indirectos que primero recopilar la información de aceptación de cliente obligatorio de sus distribuidores indirectos y, a continuación, hacer que esté disponible a Microsoft a través de la API del centro de partners o el panel del centro de partners.

**NUEVO | ¿Qué ocurre si mi cliente no acepta los términos de la MCA?**

Como partner en el programa CSP de Microsoft, eres responsable de aceptación de cliente de la MCA antes de realizar un pedido. Esto no es un requisito nuevo. Por cualquier motivo, si el cliente ya no está de acuerdo con el MCA, a continuación, el partner debe cancelar las suscripciones asociadas con este cliente.

**NUEVO | ¿Puede proporcionar un revendedor consentimiento en nombre del cliente?**

No.

**Una vez que confirmar que un cliente ha proporcionado aceptación, ¿necesito volver a confirmarla para el cliente en el futuro?**

Una vez que hayas confirmado que el cliente ha aceptado el MCA, esta información permanece en la sección de la cuenta de cliente en el centro de partners y no es necesario volver a confirmar aceptación para la duración de tiempo que el cliente está cubierto por los términos del contrato. Solo debe confirmar aceptación antes de realizar un pedido nuevo.

En el caso de que se actualiza el MCA antes de la renovación, los partners no es necesario tener el cliente volverá a obtener aceptación de cliente de la MCA inmediatamente, pero deben confirmar consentimiento para el MCA antes de la renovación. Sin embargo, puede que el cliente acepte términos nuevos antes de la renovación para sacar partido de nuevos y mejorados de los términos de la MCA.

**NUEVO | No soy cómodo lo que da a Microsoft información de contacto directo de mi cliente final. ¿Cómo usa esta información Microsoft?**

Cuando un partner crea un inquilino del cliente, es necesario para proporcionar la información de contacto principal para el cliente. Por los términos de servicios en línea, "Microsoft procesa datos personales solo en las instrucciones documentadas del cliente."

## <a href="" id="technicalfaq"></a>Técnica

**¿Cómo confirmar la aceptación del cliente y proporcionar a Microsoft los partners? ¿Las herramientas que están disponibles para admitir este proceso?**

Hay dos métodos que están disponibles a partir de 7 de agosto de 2018 para ayudar a los partners: 
1.  Los partners pueden usar la API del centro de partners para confirmar mediante programación aceptación del cliente. 
2.  Los partners pueden usar el panel del centro de partners para confirmar la aceptación del cliente.

**NUEVO | Soy un partner CSP que usa la API del centro de partners. ¿Qué cambios en la API es necesario tener en cuenta?** 

En este caso, los cambios de API para tener en cuenta son:

-   [https://docs.microsoft.com/en-us/partner-center/develop/get-confirmation-of-customer-consent](https://docs.microsoft.com/en-us/partner-center/develop/get-confirmation-of-customer-consent) 

-   [https://docs.microsoft.com/en-us/partner-center/develop/get-agreement-metadata](https://docs.microsoft.com/en-us/partner-center/develop/get-agreement-metadata)

-   [https://docs.microsoft.com/en-us/partner-center/develop/confirm-customer-consent](https://docs.microsoft.com/en-us/partner-center/develop/confirm-customer-consent)

**¿Qué ocurre si tengo dificultades para implementación de la API del centro de partners?**

Aceptación puede confirmarse manualmente mediante el panel del centro de partners.

**NUEVO | Como alternativa a usar el panel del centro de partners o la API del centro de partners, ¿de forma masiva cargar un archivo con la información de confirmación del cliente en su lugar?**

En este momento, el panel del centro de partners y la API del centro de partners son solo medio aceptado para proporcionar una confirmación de aceptación de cliente de la MCA.

**NUEVO | Soy un partner con una plataforma de aplicaciones de terceros o "software intermedio," que realiza la funcionalidad para conectar los sistemas de mi empresa con el centro de partners. ¿Qué tengo que hacer?**

Se recomienda asociados con las plataformas que se integran las API del centro de partners para hablar directamente con los proveedores respecto a sus planes y escalas de tiempo para integrar la funcionalidad de API del centro de partners que automatiza el proceso de confirmación del cliente.

**NUEVO | ¿Puede proporcionar un ejemplo de flujo de API para importar el nombre completo del cliente, la dirección de correo electrónico y la fecha de aceptación de MCA?**

Los partners se recomienda revisar la documentación de API del centro de partners para obtener más información y ejemplos.

**NUEVO | ¿Cómo se puede obtener el MCA más reciente para mostrar al cliente y usar en el registro de confirmación como parte del proceso de mi API?**

La API del centro de partners actual es muy básica y no proporciona para que los partners recuperar mediante programación una copia de la MCA ni captura las versiones del MCA aceptado por un cliente como parte de confirmación. 

**NUEVO | Con la API del centro de partners para este proceso requiere un parámetro userID. ¿Cómo administrar este requisito cuando se colocan los pedidos a través de plataformas de terceros que no se integran con Azure AD?**

En este caso, el identificador de usuario identifica al empleado en el inquilino de partner que proporciona la confirmación de consentimiento de cliente. Si el empleado no tiene una cuenta de usuario de Azure AD correspondiente en el inquilino de, a continuación, se recomienda para designar una cuenta de usuario específicas para ese propósito y crear un proceso para capturar por separado la identidad del empleado que proporciona el confirmación. 

**NUEVO | ¿Por qué es Microsoft requerir que el partner confirmar el consentimiento de cliente para el MCA en lugar de obtener directamente desde el cliente?**

El cliente compra desde el partner CSP, no directamente de Microsoft. Para usar los servicios que proporciona Microsoft, Microsoft proporciona términos que se documentan en el MCA. El MCRA requiere que los partners CSP para que los clientes revisar y aceptar el MCA antes de realizar un pedido. 

**NUEVO | Uso interfaz de usuario web de API y el panel del centro de partners para confirmar la aceptación del cliente MCA. Después de 7 de noviembre de 2018, ¿qué son las opciones para confirmar la aceptación del cliente?**

Ambas opciones están disponibles y aceptable para confirmar la aceptación del cliente MCA. Las transacciones mediante el panel del centro de partners requerirá confirmación esté presente a partir del 7 de noviembre de 2018. Las transacciones con la API del centro de partners requerirá confirmación esté presente a partir de 22 de marzo de 2019.
