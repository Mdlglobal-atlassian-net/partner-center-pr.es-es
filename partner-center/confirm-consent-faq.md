## <a href="" id="processfaq"></a>Procese

**¿Cuál es el nuevo proceso para confirmar la aceptación del cliente del contrato de Microsoft Cloud?**

Microsoft se compromete a ayudar a nuestros clientes y los partners se mantienen en esta línea. Como asociado en el programa proveedor de soluciones en la nube (CSP) de Microsoft, usted es responsable de revisar los términos y condiciones de Microsoft, incluido el uso aceptable de los productos y servicios de Microsoft con los clientes. Para ayudar a los asociados a satisfacer los requisitos de cumplimiento y garantizar la transparencia, estamos habilitando un nuevo proceso que requiere que los asociados de CSP confirmen que sus clientes han aceptado los términos del contrato de Microsoft Cloud. A partir del 7 de agosto de 2018, hemos introducido un nuevo campo en la interfaz de usuario del panel del centro de Partners, así como una API del centro de partners para que los asociados proporcionen esta información. Inicialmente, será opcional proporcionar esta confirmación; sin embargo, se requerirá a partir del 7 de noviembre de 2018. 

**¿Cuándo entra en vigor este requisito?**

Los partners pueden comenzar a confirmar la aceptación tan pronto como la funcionalidad esté disponible, el 7 de agosto de 2018. A partir del 7 de noviembre de 2018, se solicitará a los usuarios del panel del centro de partners que confirmen la aceptación del cliente del acuerdo de Microsoft Cloud en todas las transacciones nuevas. El 22 de marzo de 2019 los usuarios de la API del centro de Partners deberán hacer lo mismo. La fecha límite para los usuarios de la API se amplió para permitirle más tiempo para la implementación técnica.

**NUEVO | ¿Microsoft valida la precisión de la información del cliente proporcionada por el socio?**

Microsoft comprueba que la dirección de correo electrónico del cliente proporcionada tiene el formato correcto. Es responsabilidad del asociado proporcionar información precisa y corregir cualquier información que se encuentre inexacta. 

**NUEVO | ¿Cómo puede un asociado demostrar la aceptación del cliente del contrato de Microsoft Cloud, si está firmado, documentado, digitalizado o en papel?**

El asociado es responsable de garantizar que el cliente ha aceptado los términos del contrato de Microsoft Cloud y para administrar sus propios procesos y registros. Al usar el panel del centro de Partners o el proceso de API del centro de Partners, el asociado confirma que el cliente ha aceptado los términos del contrato de Microsoft Cloud.

**NUEVO | ¿Cómo comprobará Microsoft que el proceso de confirmación se ha completado para un cliente?**

A partir del 7 de noviembre de 2018, cuando un asociado de CSP inicia una transacción mediante el panel del centro de partners para un cliente nuevo o existente, los sistemas de Microsoft comprobarán automáticamente si la información de confirmación del cliente está presente en el centro de Partners. Consola. Si no lo está, la transacción no podrá realizarse. 

A partir del 22 de marzo de 2019, cuando un asociado de CSP inicia una transacción para un cliente nuevo o existente mediante la API del centro de Partners, los sistemas de Microsoft comprobarán automáticamente si la información de confirmación del cliente está presente. Si no lo está, la transacción no podrá realizarse. 

**NUEVO | ¿Se envía una notificación a los clientes cuya dirección de correo electrónico se proporcione en el proceso de confirmación?**

No hay ningún proceso de notificación.

## <a href="" id="policyfaq"></a>Directivas

**¿Por qué Microsoft requiere asociados para confirmar que los clientes han aceptado el contrato de Microsoft Cloud?** 

El Contrato de Microsoft Cloud protege a los partners, a los clientes y a Microsoft, proporcionando los derechos de uso del cliente, los términos y las condiciones que se aplican a los pedidos de productos y servicios de Microsoft Cloud. 

La ejecución correcta del contrato de Microsoft Cloud protege a nuestros asociados, clientes y Microsoft garantizando la alineación mutua en muchos temas importantes, entre los que se incluyen, entre otros, la seguridad, la privacidad y la protección de datos. El proceso de confirmación garantiza la transparencia y la alineación, y permite a Microsoft ayudar mejor a los clientes que necesitan responder rápidamente a las consultas regulatorias.

**¿Qué información del cliente es necesaria para completar el proceso de confirmación?**
La siguiente información es necesaria en relación con la persona que haya aceptado el contrato:
-   Fecha de aceptación del contrato
-   Nombre y apellidos
-   Dirección de correo electrónico
-   Número de teléfono (opcional) 

**NUEVO | ¿Qué ocurre si no se cumple la fecha de vencimiento?**

Los partners pueden comenzar a confirmar la aceptación tan pronto como la funcionalidad esté disponible, el 7 de agosto de 2018. A partir del 7 de noviembre de 2018, se solicitará a los usuarios del panel del centro de partners que confirmen la aceptación del cliente del acuerdo de Microsoft Cloud en todas las transacciones nuevas. El 22 de marzo de 2019 los usuarios de la API del centro de Partners deberán hacer lo mismo.

En el momento de la aplicación, no se permitirán nuevas compras para un cliente existente o cambios en el recuento de puestos en las suscripciones existentes hasta que se confirme la aceptación del cliente. Para los nuevos clientes, no se admitirán transacciones hasta que se confirme la aceptación. La renovación automática no se ve afectada por este requisito.

**¿A qué regiones se aplica este requisito?**

La confirmación de la aceptación del cliente en el contrato de Microsoft Cloud es necesaria para todas las regiones de la nube pública de Microsoft. La confirmación de la aceptación del cliente no es compatible con las nubes nacionales (como Microsoft Cloud for US Government y Microsoft Cloud Alemania).

**¿Qué asociados son responsables de confirmar la aceptación del cliente en el contrato de Microsoft Cloud?**

A partir del 7 de noviembre de 2018, los proveedores indirectos del proveedor de servicios en la nube (CSP) y los asociados de facturación directa de CSP son los responsables de confirmar la aceptación del cliente y pueden usar la API del centro de Partners o el panel del centro de partners para hacerlo. Es posible que los proveedores indirectos deban recopilar primero la información necesaria de sus revendedores indirectos y, a continuación, ponerla a disposición de Microsoft a través de la API del centro de Partners o el panel del centro de Partners. Contractualmente, es el proveedor indirecto que tiene un contrato de enlace con Microsoft para garantizar que los clientes finales de sus revendedores de CSP hayan aceptado el contrato de Microsoft Cloud. 

**¿Qué escenarios de cliente requieren la confirmación de la aceptación del cliente del contrato de Microsoft Cloud?**

La aceptación del contrato de Microsoft Cloud debe confirmarse cuando los asociados de CSP quieren realizar las siguientes transacciones:
1.  Un nuevo cliente completa las compras nuevas
2.  Un cliente existente completa las compras nuevas
3.  Un cliente existente actualiza el número de puestos a las suscripciones existentes

**¿Puedo confirmar la aceptación de los clientes existentes independientes de sus nuevas compras o incorporaciones de puestos?**

Sí. Se puede confirmar la aceptación del cliente para todos los clientes incluidos en el contrato de Microsoft Cloud.

**¿Cuáles son los requisitos de los clientes que tienen varios asociados de CSP?**

La confirmación de la aceptación la confirma el partner. Si un cliente realiza transacciones con varios partners del CSP, se requerirá cada partner que confirme la aceptación.

**Soy un revendedor indirecto de CSP que trabaja con proveedores indirectos de CSP. ¿Qué es mi responsabilidad?**

Contractualmente, es el proveedor indirecto que tiene un contrato de enlace con Microsoft para asegurarse de que el cliente final ha aceptado el contrato de Microsoft Cloud. Sin embargo, los proveedores indirectos pueden tener acuerdos independientes con revendedores indirectos para recoger las aceptaciones de los clientes. En este caso, los proveedores indirectos necesitarán primero recopilar la información de la aceptación del cliente de sus revendedores indirectos y, a continuación, ponerla a disposición de Microsoft a través de la API del Centro de partners o el Panel del Centro de partners. Puedes apoyar este proceso proporcionando rápidamente la información necesaria a los proveedores indirectos, conforme la pidan.

**Soy un proveedor indirecto de CSP. ¿Es necesario coordinar el revendedor indirecto de CSP para que esté preparado?**

Se recomienda a los proveedores indirectos que necesiten recopilar la información de aceptación del cliente de sus revendedores indirectos que comuniquen inmediatamente este requisito a sus revendedores indirectos, para así coordinar la entrega y la recepción de esta información.

**¿Cómo puedo explicar a mis clientes que deben confirmar su aceptación del contrato de Microsoft Cloud?**

La ejecución correcta del contrato de Microsoft Cloud protege a Microsoft, a sus asociados y clientes al garantizar la alineación mutua en muchos temas importantes, incluidos, entre otros, la seguridad, la privacidad y la protección de datos. El proceso de confirmación garantiza la transparencia y la alineación, y permite a Microsoft ayudar mejor a los clientes que necesitan responder rápidamente a las consultas regulatorias.

**NUEVO | Si un cliente trabaja con un proveedor indirecto de CSP único, pero las transacciones se completan con varios distribuidores indirectos de CSP, ¿es necesario que cada revendedor indirecto de CSP proporcione al proveedor indirecto de CSP la información de confirmación?**

El proveedor indirecto de CSP es responsable de proporcionar la confirmación de la aceptación del cliente de Microsoft Cloud acuerdo a Microsoft. Para cada cuenta de cliente, solo se puede registrar una confirmación en el panel del centro de Partners al mismo tiempo. 

El proveedor indirecto de CSP es responsable de confirmar que una persona de contacto del cliente responsable de aceptar el contrato de Microsoft Cloud lo ha hecho, independientemente del número de revendedores indirectos de CSP con los que se han realizado las transacciones.

**NUEVO | ¿Quién debe ser la persona autorizada para confirmar la aceptación del contrato de Microsoft Cloud dentro de la organización del cliente?**

Es responsabilidad del asociado obtener la aceptación del cliente del contrato de Microsoft Cloud de una persona dentro de la organización del cliente. Es responsabilidad del cliente determinar quién está autorizado para proporcionar aceptación. 

**NUEVO | ¿Qué versión del acuerdo de Microsoft Cloud son los asociados responsables de confirmar la aceptación del cliente? ¿Los asociados deben indicar qué versión del acuerdo de Microsoft Cloud fue aceptada por el cliente?**

Los asociados son responsables del uso del contrato de Microsoft Cloud más actual al crear transacciones de CSP con un nuevo cliente. Los asociados deben confirmar la aceptación del contrato de Microsoft Cloud más actual al renovar, cambiar o completar nuevas transacciones con un cliente existente. 

En el caso de que el acuerdo de Microsoft Cloud se actualice antes de la renovación, no es necesario que el cliente vuelva a obtener la aceptación del cliente del contrato de Microsoft Cloud inmediatamente, sino que debe volver a confirmar el consentimiento del acuerdo de Microsoft Cloud antes para la renovación. Sin embargo, puede que desee que el cliente acepte nuevos términos antes de la renovación para aprovechar las ventajas de los términos nuevos y mejorados del contrato de Microsoft Cloud.

**NUEVO | Soy un revendedor indirecto de CSP. Si recordo la aceptación del cliente del contrato de Microsoft Cloud en los registros de la empresa, ¿necesito proporcionar también esta información a través del panel del centro de Partners o de la API del centro de Partners?**

Contractualmente, es el proveedor indirecto que tiene un contrato de enlace con Microsoft para asegurarse de que el cliente final ha aceptado el contrato de Microsoft Cloud. Sin embargo, los proveedores indirectos pueden tener acuerdos independientes con revendedores indirectos para recoger las aceptaciones de los clientes. En este caso, los proveedores indirectos necesitarán primero recopilar la información de la aceptación del cliente de sus revendedores indirectos y, a continuación, ponerla a disposición de Microsoft a través de la API del Centro de partners o el Panel del Centro de partners.

**NUEVO | ¿Qué ocurre si mi cliente no acepta los términos del contrato de Microsoft Cloud?**

Como asociado en el programa CSP de Microsoft, usted es responsable de la aceptación por parte del cliente del contrato de Microsoft Cloud antes de realizar un pedido. No se trata de un requisito nuevo. Por cualquier motivo, si el cliente ya no está de acuerdo con el contrato de Microsoft Cloud, el asociado debe cancelar las suscripciones asociadas a este cliente.

**NUEVO | ¿Puede un reseller dar su consentimiento en nombre del cliente?**

No.

**Una vez que se confirma que un cliente ha proporcionado la aceptación, ¿es necesario volver a confirmarla para el cliente en el futuro?**

Una vez que haya confirmado que el cliente ha aceptado el contrato de Microsoft Cloud, esta información permanece en la sección de la cuenta de cliente del centro de Partners y no es necesario reconfirmar la aceptación durante el tiempo que está incluido el cliente. términos del acuerdo. Solo debe volver a confirmar la aceptación antes de colocar un nuevo pedido.

En el caso de que el acuerdo de Microsoft Cloud se actualice antes de la renovación, no es necesario que el cliente vuelva a obtener la aceptación del cliente del contrato de Microsoft Cloud inmediatamente, sino que debe volver a confirmar el consentimiento del acuerdo de Microsoft Cloud antes para la renovación. Sin embargo, puede que desee que el cliente acepte nuevos términos antes de la renovación para aprovechar las ventajas de los términos nuevos y mejorados del contrato de Microsoft Cloud.

**NUEVO | No me encuentro en ofrecer información de contacto directo de Microsoft My end Customer. ¿Cómo usará Microsoft esta información?**

Cuando un socio crea un inquilino de cliente, es necesario proporcionar una información de contacto principal para el cliente. Según los términos de los servicios en línea, "Microsoft procesará los datos personales solo en instrucciones documentadas del cliente".

## <a href="" id="technicalfaq"></a>T

**¿Cómo confirman los asociados la aceptación del cliente y los proporcionan a Microsoft? ¿Qué herramientas están disponibles para admitir este proceso?**

A partir del 7 de agosto de 2018, hay dos métodos disponibles para ayudar a los asociados: 
1.  Los partners pueden usar la API del Centro de partners para confirmar programáticamente la aceptación del cliente. 
2.  Los asociados pueden usar el panel del centro de partners para confirmar la aceptación del cliente.

**NUEVO | Soy un asociado de CSP que usa la API del centro de Partners. ¿Qué cambios de API se deben tener en cuenta?** 

En este caso, los cambios de la API para la cuenta de son:

-   [https://docs.microsoft.com/partner-center/develop/get-confirmation-of-customer-consent](https://docs.microsoft.com/partner-center/develop/get-confirmation-of-customer-consent) 

-   [https://docs.microsoft.com/partner-center/develop/get-agreement-metadata](https://docs.microsoft.com/partner-center/develop/get-agreement-metadata)

-   [https://docs.microsoft.com/partner-center/develop/confirm-customer-consent](https://docs.microsoft.com/partner-center/develop/confirm-customer-consent)

**¿Qué ocurre si tengo dificultades para implementar la API del centro de Partners?**

La aceptación puede confirmarse manualmente a través del Panel del Centro de partners.

**NUEVO | Como alternativa al uso del panel del centro de Partners o de la API del centro de Partners, ¿puedo realizar una carga masiva de un archivo con la información de confirmación del cliente?**

En este momento, el panel del centro de Partners y la API del centro de Partners son los únicos medios aceptados para proporcionar la confirmación de la aceptación del cliente del contrato de Microsoft Cloud.

**NUEVO | Soy un asociado que usa una plataforma de terceros, o "middleware", que lleva a cabo la funcionalidad de conexión de los sistemas de la empresa con el centro de Partners. ¿Qué tengo que hacer?**

Es aconsejable que los asociados que usan plataformas que integran la API del centro de Partners hablen directamente con los proveedores con respecto a sus planes y escalas de tiempo para integrar la funcionalidad de la API del centro de partners que automatiza el proceso de confirmación del cliente.

**NUEVO | ¿Puede proporcionar un ejemplo de flujo de API para importar el nombre completo del cliente, la dirección de correo electrónico y la fecha de la aceptación del acuerdo de Microsoft Cloud?**

Se recomienda que los asociados revisen la documentación de la API del centro de partners para obtener más información y ejemplos.

**NUEVO | Cómo obtener el contrato de Microsoft Cloud más actual para mostrar al cliente y usarlo en el registro de confirmación como parte de mi proceso de API?**

La API del centro de Partners actual es muy básica y no permite que los asociados recuperen mediante programación una copia del acuerdo de Microsoft Cloud, ni tampoco capturan versiones del acuerdo de Microsoft Cloud aceptado por un cliente como parte de la confirmación. 

**NUEVO | El uso de la API del centro de partners para este proceso requiere un parámetro userID. Cómo administrar este requisito cuando los pedidos se realizan a través de plataformas de terceros que no se integran con Azure AD?**

En este caso, el userID identifica al empleado en el inquilino del asociado que proporciona la confirmación del consentimiento del cliente. Si el empleado no tiene una cuenta de usuario de Azure AD correspondiente en el inquilino del asociado, se recomienda designar una cuenta de usuario específica para ese fin y crear un proceso para capturar por separado la identidad del empleado que proporciona el confirmación. 

**NUEVO | ¿Por qué Microsoft requiere que el socio firme el consentimiento del cliente en el contrato de Microsoft Cloud en lugar de obtenerlo directamente del cliente?**

El cliente está comprando a través del asociado de CSP, no directamente de Microsoft. Para usar los servicios que proporciona Microsoft, Microsoft proporciona términos que se documentan en el contrato de Microsoft Cloud. El contrato de revendedor de Microsoft Cloud requiere que los clientes de CSP revisen y acepten el contrato de Microsoft Cloud antes de realizar un pedido. 

**NUEVO | Utilizo la interfaz de usuario Web del panel del centro de Partners y API para confirmar la aceptación del cliente del acuerdo de Microsoft Cloud. Después del 7 de noviembre de 2018, ¿cuáles son mis opciones para confirmar la aceptación del cliente?**

Ambas opciones están disponibles y son aceptables para confirmar la aceptación del cliente de Microsoft Cloud acuerdo. Las transacciones que usan el panel del centro de Partners requerirán que se presente la confirmación a partir del 7 de noviembre de 2018. Las transacciones que usan la API del centro de Partners requerirán que se presente la confirmación a partir del 22 de marzo de 2019.
